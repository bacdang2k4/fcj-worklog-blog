**On this page**, you will need to introduce your worklog. **How** did you complete it? How many weeks did you take to complete the program? **What** did you do in those weeks?

Typically, and as a standard, a worklog is carried out over about 3 months (throughout the internship period) with weekly contents as follows:

**Week 1:** [Getting familiar with AWS and basic AWS services](1.1-week1/)

**Week 2:** [Learn about VPC in AWS](1.2-week2/)

**Week 3:** [Master the basics of EC2](1.3-week3/)

**Week 4:** [Explore AWS storage services and get familiar with the AWS CLI command system](1.4-week4/)

**Week 5:** [Learn and practice AWS security content (IAM, Cognito, Organization, Identity Center, KMS, Security Hub)](1.5-week5/)

**Week 6:** [Learn Microservice Architecture, learn gRPC and Apache Kafka, review for midterms, and complete a group project system architecture drawing.](1.6-week6/)

**Week 7:** [Learn AWS Lambda, install and use AWS CLI, practice deploying Java functions to Lambda and building APIs with API Gateway](1.7-week7/)

**Week 8:** [Complete a group proposal, review comprehensive AWS knowledge (security, architecture, performance, costs), and take the midterm exam](1.8-week8/)

**Week 9:** [Learn AWS IoT Core and AWS Amplify to expand your knowledge of the AWS ecosystem and serve your project](1.9-week9/)

**Week 10:** [Learned Terraform and GitHub Actions while deploying the basic project's infrastructure architecture](1.10-week10/)

**Week 11:** [Deploy entire AWS infrastructure using Terraform and build GitHub Actions pipeline for the system](1.11-week11/)

**Week 12:** [Complete group project demo, prepare presentation slides and compile personal worklog](1.12-week12/)
